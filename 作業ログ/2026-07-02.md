#### 音声入力不具合の原因調査

`adb logcat`で該当箇所を確認:
```
SodaSpeechRecognizer: Failed to get language pack of required locale: error 13
RecognitionClient: #onStartOfSpeech
RecognitionClient: #onRecognitionFinished no speech - erroring
RecognitionServiceImpl: Speech recognition error type NO_SPEECH_DETECTED
```
- アプリ側（`speech_to_text`）は正常に音声認識セッションを開始できている
- 「発話が検出されない（NO_SPEECH_DETECTED）」で終了しており、**Androidエミュレータの仮想マイクがデフォルトではPCの実マイク音声を拾わない設定になっている**ことが原因の可能性が高い（アプリ側のバグではなくエミュレータ環境固有の制約と推測）
- 対策候補（ユーザーと相談し、今回は見送り。次回実機テスト時に改めて確認予定）:
  - エミュレータのExtended Controls → Microphone → "Virtual microphone uses host audio input" を有効化して再テスト
  - 実機（USB接続）での確認に切り替える

### 6. 実機（USB）接続の試行 → 断念、エミュレータでの音声入力問題を解決

- コンビニで購入したUSBケーブルで実機接続を試みたが、`adb devices`にもWindowsのデバイス一覧にも認識されず
  - 通知バーにUSB接続の通知が出ない症状から、**データ転送非対応（充電専用）のケーブル**の可能性が高いと判断
  - データ対応ケーブルの調達は次回以降に持ち越し、今回はエミュレータでの確認に切り替え
- 前回保留していた「音声入力がNO_SPEECH_DETECTEDで失敗する」問題を再調査
  - エミュレータのExtended Controls → Microphone パネルで、「Virtual microphone uses host audio input」に相当する設定は **「Enable Host Microphone Access」** という名称だった（このAVD/エミュレータバージョンでの表記）
  - これをONにしたところ、**音声入力が正常に動作することを確認**
    - 「テスト」と発話 → 確認画面のタイトル欄に「テスト」が正しく入力される
    - 日時を含む発話（例:「ズンバ」＋日時指定） → 「ズンバ / 7月4日(土) 13:15」として保存され、カレンダーの7月4日（土）にイベントドットが表示
  - これにより、**音声入力→日本語日時パーサー→確認画面→保存→一覧・カレンダー反映、の一連のフローがエミュレータ上で完全に動作することを確認**
  - 結論: 音声入力の不具合はアプリ側の問題ではなく、エミュレータのマイク設定（デフォルトOFF）が原因だった
