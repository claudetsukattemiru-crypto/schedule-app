## やったこと

1. **方針確認**
   - Flutterモバイルアプリの計画はそのまま維持しつつ、別途お試し用のWeb版（HTML/CSS/JavaScript）も並行して作りたいという要望を確認
   - Web版は `index.html` / `style.css` / `script.js` の3ファイル構成、技術はWeb Speech API + localStorageに決定

2. **Web版プロトタイプ作成**（`web-prototype/` フォルダ、Flutter版とは独立）
   - `index.html`: マイクボタン、確認/編集フォーム（音声入力・手動追加で共通）、予定一覧
   - `style.css`: モバイル幅を想定したシンプルなスタイル
   - `script.js`:
     - Web Speech API（`ja-JP`）で音声→テキスト化
     - Flutter版プランと同じ設計思想のルールベース日本語日時パーサー（絶対日付→相対日→週+曜日→単独曜日、明示時刻→曖昧時刻語、の順で順序付きマッチャーを適用）をJavaScriptに移植
     - 解析結果は確認画面に表示し、**音声からの自動保存はしない**（手動修正してから保存する設計はFlutter版の方針を踏襲）
     - データはlocalStorageに保存（id/title/startAt/isAllDay/sourceType/rawTranscript/createdAt/updatedAt）。本番Flutter版と異なり、これは使い捨てプロトタイプなのでUUID同期用カラム（deletedAt等）は省略

3. **動作確認**
   - パーサーのロジックをNode.jsで単体テスト（プラン記載の代表例「来週火曜の15時に会議」等）→ すべて期待通りに解析できることを確認
   - `file://` で開くと音声認識がうまく動かなかったため、Node.jsで簡易HTTPサーバーを立てて `http://localhost:8000` 経由で動作確認するよう変更
   - 「no-speech」エラーが発生 → 原因切り分けの結果、**Windowsの既定の音声入力デバイスが「ジャックマイク」（外部マイク端子用、何も挿していない）になっていた**ことが原因と判明。既定入力を内蔵「マイク」に切り替えて解決
   - 音声認識→解析→確認画面→保存、の一連の流れが動作することを確認。認識結果が誤っていた場合も確認画面でタイトルを手動修正して保存できることを確認済み

4. **月表示カレンダーの追加**（自前実装、外部ライブラリ不使用）
   - `index.html` に月送り（‹/›）・今日ボタン・曜日見出し・カレンダーグリッドを追加
   - `script.js`: `renderCalendar()` で日曜始まりの月グリッドを描画。今日の日付は枠線、予定がある日はドット表示、日付クリックで下の一覧をその日だけにフィルタ（「すべて表示」で解除）
   - 予定の追加・編集・削除のたびにカレンダーと一覧を両方再描画する `renderAll()` に統一

5. **音声入力の聞き取り時間を延長**

   - マイクボタンをもう一度押すと聞き取り終了、という明示的な操作に変更（押し忘れ対策として最大30秒の自動終了タイマーも追加）

6. **「場所」項目を追加**

7. **カレンダーの配色調整**（ユーザー希望を反映しながら数回イテレーション）
